# Plan 4.3: Interviewer Dashboard

## Objective
Build the main dashboard for the interviewer to trigger the AI analysis and view the generated brief.

## Context
- frontend/src/App.jsx
- frontend/src/services/api.js

## Tasks

<task type="auto">
  <name>Build Trigger Form</name>
  <files>
    - frontend/src/pages/InterviewerSetup.jsx
  </files>
  <action>
    - Create a form capturing `companyName` and optional `companyUrl`.
    - On submit, call `api.triggerPipeline()`.
    - On success, use `useNavigate` to push to `/session/{executionId}`.
    - Style with Tailwind using maroon buttons and input borders.
  </action>
  <verify>cat frontend/src/pages/InterviewerSetup.jsx</verify>
  <done>Component captures input and redirects to the session page.</done>
</task>

<task type="auto">
  <name>Build Session Polling Dashboard</name>
  <files>
    - frontend/src/pages/InterviewerDashboard.jsx
    - frontend/src/components/BriefView.jsx
  </files>
  <action>
    - In `InterviewerDashboard`, extract `id` from URL params.
    - Set up a `useEffect` interval that polls `api.getSession(id)` every 5 seconds WHILE `status` is `CREATED` or `ANALYSIS_COMPLETE`.
    - Stop polling if `status` is `READY` or `FEEDBACK_RECEIVED`.
    - Build `BriefView.jsx` to render the `interviewerBrief` (Company Profile, Interview Strategy, Questions) and `analysisResults` (Entities/Key phrases) into a clean, desktop-first layout.
  </action>
  <verify>cd frontend && npm run build</verify>
  <done>Dashboard renders the structured AI output and handles polling states.</done>
</task>

## Success Criteria
- [ ] Pipeline triggers successfully from UI.
- [ ] UI automatically polls API Gateway and updates when step functions complete.
- [ ] AI intelligence data is formatted readably.
