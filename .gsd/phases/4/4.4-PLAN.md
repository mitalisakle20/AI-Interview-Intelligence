# Plan 4.4: Interviewee Portal & Web Form

## Objective
Implement the interviewee-facing UI where they can review the AI findings, correct inaccuracies, and select questions they want to discuss.

## Context
- frontend/src/App.jsx
- frontend/src/services/api.js

## Tasks

<task type="auto">
  <name>Build Interviewee Profile View</name>
  <files>
    - frontend/src/pages/IntervieweePortal.jsx
    - frontend/src/components/IntervieweePacket.jsx
  </files>
  <action>
    - Extract `id` from URL params (`/interviewee/:id`).
    - On mount, `api.getSession(id)`. If status is `FEEDBACK_RECEIVED`, show a "Thank you" state and disable the form.
    - Build `IntervieweePacket.jsx` to render the AI-generated company profile, recent news, and strengths/weaknesses (from `session.intervieweePacket`).
    - Above the profile, add a polite header explaining that the AI prepared this and asking them to review it for accuracy.
  </action>
  <verify>cd frontend && npm run build</verify>
  <done>Interviewee portal successfully loads session data and displays the AI packet.</done>
</task>

<task type="auto">
  <name>Build Feedback Form</name>
  <files>
    - frontend/src/components/FeedbackForm.jsx
    - frontend/src/pages/IntervieweePortal.jsx
  </files>
  <action>
    - Create a form with a text area for "Corrections / Clarifications" (optional).
    - Map over `session.intervieweePacket.suggested_questions` and render them as checkboxes so the interviewee can select exactly 2 or 3 questions.
    - Implement validation: Ensure at least 1 and no more than 3 questions are selected before enabling the submit button.
    - On submit, call `api.submitFeedback(id, corrections, selectedQuestions)`.
    - Handle loading state and display a success message upon completion.
  </action>
  <verify>cat frontend/src/components/FeedbackForm.jsx</verify>
  <done>Feedback form captures input, enforces selection limits, and POSTs to the backend.</done>
</task>

## Success Criteria
- [ ] Read-only view of the AI profile works.
- [ ] Question selection enforces validation (max 3).
- [ ] Form submission successfully calls the new feedback endpoint.
