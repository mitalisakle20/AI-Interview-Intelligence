# Plan 4.1: Backend Feedback & PDF Endpoints

## Objective
Implement the two new AWS Lambda endpoints needed to support the Two-Stage Workflow and Client-Side PDF Generation.

## Context
- .gsd/SPEC.md
- .gsd/ROADMAP.md
- backend/template.yaml

## Tasks

<task type="auto">
  <name>Implement SubmitFeedbackFunction</name>
  <files>
    - backend/functions/submit_feedback/handler.py
    - backend/template.yaml
  </files>
  <action>
    - Create `submit_feedback/handler.py` with `lambda_handler(event, context)`.
    - It should parse `sessionId` from path parameters and `body` containing `corrections` (str) and `selectedQuestions` (list).
    - Use `DynamoDBService().update_session(session_id, session["createdAt"], {"feedback": body})` to patch the DB.
    - Add `SubmitFeedbackFunction` to `template.yaml` bound to `POST /session/{sessionId}/feedback` with `DynamoDBCrudPolicy`.
  </action>
  <verify>sam build SubmitFeedbackFunction</verify>
  <done>Lambda handler exists and template.yaml includes the API route for POST feedback.</done>
</task>

<task type="auto">
  <name>Implement GeneratePdfFunction</name>
  <files>
    - backend/functions/generate_pdf/handler.py
    - backend/template.yaml
    - backend/requirements.txt
  </files>
  <action>
    - Add `xhtml2pdf` to `backend/requirements.txt`.
    - Create `generate_pdf/handler.py`. Extract `html_content` from the request body.
    - Use `xhtml2pdf.pisa.CreatePDF` to render the HTML string into a `io.BytesIO()` buffer.
    - Return a 200 response where `body` is the base64-encoded PDF buffer, and `isBase64Encoded: True`. Headers should include `Content-Type: application/pdf` and `Content-Disposition: attachment; filename=document.pdf`.
    - Add `GeneratePdfFunction` to `template.yaml` bound to `POST /pdf`.
    - Set the API Gateway endpoint binary media types in template.yaml to include `application/pdf` if necessary, or just rely on the base64 proxy integration.
  </action>
  <verify>sam build GeneratePdfFunction</verify>
  <done>PDF Lambda returns base64 encoded PDF from an HTML string.</done>
</task>

## Success Criteria
- [ ] `POST /session/{id}/feedback` endpoint is defined in SAM.
- [ ] `POST /pdf` endpoint is defined in SAM.
- [ ] `xhtml2pdf` is added to shared runtime layer.
